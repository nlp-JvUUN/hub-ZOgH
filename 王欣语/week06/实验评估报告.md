# 文本分类实验评估报告

> 生成时间：2026-05-29
> 实验项目：BERT 文本分类 vs LLM 分类对比

---

## 一、实验概览

| 实验              | 模型                  | 策略           | 训练数据     | 轮数     |
| :---------------- | :-------------------- | :------------- | :----------- | :------- |
| BERT-cls          | bert-base-chinese     | CLS 池化       | 1000条       | 2 epoch  |
| BERT-mean         | bert-base-chinese     | Mean 池化      | 1000条       | 2 epoch  |
| BERT-max          | bert-base-chinese     | Max 池化       | 1000条       | 2 epoch  |
| BERT-cls-weighted | bert-base-chinese     | CLS + 类别权重 | 全部(53K)    | 3 epoch  |
| LLM-zero-shot     | Qwen2-0.5B-Instruct   | Zero-shot      | 无           | -        |
| LLM-SFT           | Qwen2-0.5B-Instruct   | LoRA 微调      | 100条        | 1 epoch  |

---

## 二、BERT 三种池化策略对比

| 指标                 | CLS       | Mean      | Max       |
| :------------------- | :-------- | :-------- | :-------- |
| Epoch 1 val_acc      | 50.2%     | 51.5%     | 46.4%     |
| Epoch 2 val_acc      | 51.3%     | 52.1%     | 51.7%     |
| Epoch 2 val_macro_f1 | 0.480     | 0.490     | 0.466     |
| 训练时间/epoch       | ~82s      | ~92s      | ~99s      |

### 结论
- **Mean 池化** 验证集表现最好（val_acc 52.1%，macro_f1 0.490）
- **CLS 池化** 训练最快，但略低于 mean
- **Max 池化** 初期收敛慢（epoch1 仅 46.4%），但最终追赶上

---

## 三、类别权重（Class Weight）效果

| 指标                 | CLS (无权重) | CLS + 权重 |
| :------------------- | :----------- | :--------- |
| 训练数据             | 1000条       | 全部 53K条 |
| 最终 val_acc         | 51.3%        | 55.9%      |
| 最终 val_macro_f1    | 0.480        | 0.553      |
| 训练时间/epoch       | ~82s         | ~339s      |

### 结论
- **使用类别权重 + 全部数据**，准确率提升 **4.6%**
- **Macro F1 提升显著**（0.480 → 0.553），说明对不均衡类别效果更好
- 训练时间增加是因为数据量从 1000 → 53000 条

---

## 四、LLM 对比（Qwen2-0.5B-Instruct）

| 指标                 | Zero-shot   | SFT (LoRA微调) |
| :------------------- | :---------- | :------------- |
| 准确率               | 20.0% (1/5) | 40.0% (2/5)    |
| 无法解析             | 3/5 (60%)   | 0/5 (0%)       |
| 训练数据             | 无          | 100条          |
| 可训练参数           | -           | 仅 0.22%       |

### 结论
- **SFT 后准确率翻倍**（20% → 40%）
- **Zero-shot 大量输出无法匹配类别**（如"房地产"、"武器"、"宗教"）
- **SFT 后输出更规范**，全部能匹配到预定义类别
- LoRA 仅训练 1M 参数，高效且省显存

## 五、综合排名

| 排名 | 方法                  | 验证准确率 | 特点                  |
| :--- | :-------------------- | :--------- | :-------------------- |
| 1    | BERT + CLS + 权重     | 55.9%      | 数据充足时的最佳选择  |
| 2    | BERT + Mean           | 52.1%      | 小数据量时最稳定      |
| 3    | BERT + Max            | 51.7%      | 收敛慢但最终效果接近  |
| 4    | BERT + CLS            | 51.3%      | 训练最快，baseline    |
| 5    | LLM + SFT             | 40.0%      | 生成式，输出灵活      |
| 6    | LLM + Zero-shot       | 20.0%      | 无需训练，但效果差    |

---

## 六、关键发现

1. **池化策略**：Mean > CLS > Max（小数据量下）
2. **类别权重**：对不均衡数据效果显著，Macro F1 提升 15%
3. **数据量**：1000条 vs 53000条，准确率差距约 4-5%
4. **LLM 不适合直接做分类**：Zero-shot 效果很差，SFT 后仍不如 BERT
5. **推理示例**（"苹果发布 iPhone 17"）：
   - 三种池化都正确预测「科技」
   - Mean 置信度最高（83.4%），Max 最低（66.1%）

---

## 七、建议

| 场景                  | 推荐方案                        |
| :-------------------- | :------------------------------ |
| 追求最高准确率        | BERT + 类别权重 + 全部数据      |
| 快速实验/CPU 环境     | BERT + Mean，1000条数据         |
| 资源极度受限          | BERT + CLS，最小配置            |
| 需要生成式输出        | LLM + LoRA SFT（但准确率较低）  |
