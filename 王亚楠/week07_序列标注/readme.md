NER：bert+crf
chinese bert 在tokennization时做过特殊的处理，用的不是类似BPE的统计方法，而是以字符为粒度进行token化，
现在的大模型因为语料都是多语言混合所以tokennization 使用的是BPE的子词粒度的统计方法，所以在做命名实体识识别任务时让模型输出实体的位置是做不到的 
任务定义
给定一个输入序列，为序列中的每一个元素分配一个标签，输出序列与输入序列等长。
应用：中文分词、词性标注（POS）、命名实体识别（NER）
中文分词
任务定义：将连续汉字序列切分为有意义的词语单元，是中文 NLP 的基础预处理步骤。

词性标注
任务定义：为句子中每个词（或字）分配其在该上下文中的语法类别（词性），是句法分析的基础。
常见词性标签（PKU 标注体系）
[图片]
与分词的关系：
  - 实际系统中常联合建模（分词+词性同步输出）
  - 词性为下游任务提供结构化特征
  - BERT 时代：词性标注可作为序列标注微调任务
分词  →  词性标注  →  NER / 句法分析（典型 NLP 流水线前处理）大模型时代就不需要这个流程了，但对于具体的小任务用Bert的话还需要这样做

框架本质：
NER 模型学习的是：给定上下文，判断一段连续 token 是否属于目标类别。这个能力与[类别叫什么名字]无关。只要提供足够的标注样本，模型就能识别任意类型的 span。类似图像中的目标检测

标注体系设计要点
标注方案选择
[图片]
设计原则：
  - 类别数不宜过多（>20 类会显著增加难度）
  - 类别间边界要清晰，标注歧义越少越好
  - 先用 BIO，数据充足后再考虑 BIOES
  - 保持标注一致性：双人标注 + 仲裁
嵌套实体处理：
文本：北      京      大      学 
外层：B-ORG   I-ORG   I-ORG   I-ORG
内层：B-LOC   I-LOC
应对方案：
  - 多层标注（每层独立 BIO 序列）
  - Span-based 方法（枚举所有 span 打分）
  - 业务中若嵌套比例低，可忽略处理

序列标注的其他 NLP 应用
1. 语义角色标注（SRL）
  - 标注谓词-论元结构：
     谁(ARG0) 对谁(ARG1) 做了什么(V)
  示例：[马云]A0 [创立]V [阿里巴巴]A1
2. 依存句法分析（简化视角）
  - 可将 Head 预测转化为序列标注：
  每个词标注其 Head 词的位置偏移量
  → 简化版 Arc-Eager 解析器
3. 关键词 / 短语抽取
  - 将关键词标注为 B/I，非关键词为 O
  输出等长标签 → 抽取关键 span
  也可结合置信度排序输出 Top-K
4. 事件抽取（Event Extraction）
  - 触发词识别：找到事件的核心动词/名词
  论元抽取：标注参与者、时间、地点等角色
  本质也是 多类别序列标注
共同本质：定义标注体系  →  标注训练数据  →  训练序列标注模型  →  解码输出
