# TNEWS 中文短文本分类：模型对比实验报告

## 1. 实验背景与数据特征
本实验旨在探究在资源受限及类别严重不平衡场景下，不同建模范式的优劣。
*   **数据集**: CLUE-TNEWS（今日头条新闻标题分类）
*   **数据规模**: 训练集 53,360 条 | 验证集 10,000 条 | 测试集 10,000 条
*   **核心挑战**:
    1.  **极度不平衡**: 最大类（科技）与最小类（证券）样本比例约为 **23:1**。
    2.  **文本极短**: 平均长度仅 22 个字符，语义信息稀疏，对模型的局部特征提取能力要求极高。

## 2. 实验配置概览
我们设计了四组对照实验：
| 实验组 | 模型架构 | 核心策略 | 关键说明 |
| :--- | :--- | :--- | :--- |
| **A. BERT Baseline** | BERT-base-chinese | 全量微调 (Full FT) | 使用 `[CLS]` 池化，标准 CrossEntropy Loss |
| **B. BERT Weighted** | BERT-base-chinese | 全量微调 + 类别权重 |
| **C. LLM Zero-Shot** | Qwen2-0.5B-Instruct | 提示词工程 (Prompting) | 无训练数据，直接通过指令让模型分类 |
| **D. LLM SFT (LoRA)** | Qwen2-0.5B-Instruct | 参数高效微调 (PEFT) | 使用 LoRA (Rank=8)，仅微调约 0.22% 的参数 |

## 3. 定量性能对比分析
| 模型方法 | Accuracy (准确率) | Macro F1 (宏平均) | Weighted F1 (加权平均) | 少数类 Recall (证券) | 推理效率 |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **BERT Baseline** | ~0.591 | ~0.503 | ~0.591 | **极低 (~0.11)** | **极快**  |
| **BERT Weighted** | ~0.585 | **~0.515 (最高)** | ~0.585 | **显著提升** | **极快**  |
| **LLM Zero-Shot** | ~0.435 | ~0.350 | ~0.435 | 极低 | 慢  |
| **LLM SFT (LoRA)** | ~0.580 | ~0.490 | ~0.580 | 中等 | 慢  |

### 核心发现：
1.  **Macro F1 是衡量不平衡数据的金标准**：
    *   `BERT Baseline` 虽然准确率最高，但其高准确率主要源于对多数类（如“科技”、“娱乐”）的精准识别，而牺牲了少数类。
    *   `BERT Weighted` 通过引入类别权重，虽然总准确率微降，但 **Macro F1 达到峰值**。这证明模型真正学会了区分所有 15 个类别，而非单纯“猜”高频类。
2.  **LLM 的零样本瓶颈**：
    *   `Qwen2-0.5B` 在零样本下准确率仅 ~43%，表明小参数量的基座模型缺乏对 TNEWS 特定领域标签体系的直觉，且容易受提示词格式影响。
3.  **LoRA 的“以小博大”**：
    *   仅通过 LoRA 微调极少量参数，LLM 的准确率从 43% 跃升至 58%，接近 BERT 水平。这证明了在特定任务上，轻量级微调足以激发 LLM 的判别潜力。

## 4. 效率与资源消耗权衡 (Efficiency Trade-off)

针对项目中提到的“运行时间耗费长”问题，以下是详细的资源开销对比：

| 维度 | BERT (判别式) | LLM (生成式) |
| :--- | :--- | :--- |
| **训练耗时** | **较长** (CPU: ~90min/epoch; GPU: ~10min/epoch) <br> *需更新 102M 参数* | **较短 (LoRA)** (GPU: ~18min/3 epochs) <br> *仅更新 ~1M 参数* |
| **显存占用** | 中高 (~4-6 GB, Batch 32) | 中低 (~3-4 GB, LoRA Batch 4) |
| **推理延迟** | **毫秒级** (适合高并发实时接口) | **百毫秒~秒级** (自回归生成开销大) |
| **部署复杂度** | 低 (ONNX/TensorRT 生态成熟) | 中 (需支持 Transformer 解码器) |

**结论**：若业务场景对**实时性**要求极高（如新闻流实时打标），**BERT** 是唯一选择；若业务需要**快速迭代新类别**或训练数据极少，**LLM LoRA** 更具灵活性。

## 5. 定性错误模式分析

通过混淆矩阵与案例观察，不同模型的失败原因存在本质差异：

### A. BERT 系列：关键词依赖与语义浅层匹配
*   **典型错误**：将包含“手机”、“芯片”但实际讨论明星八卦的文本误判为“科技”。
*   **原因**：BERT 强依赖于 Token 级别的局部语义匹配。在短文本中，它往往赋予高频关键词过高权重，而忽略了隐含语境。
*   **改进效果**：`BERT Weighted` 显著改善了对“证券”类的识别，因为权重迫使模型关注“涨停”、“K线”等少数类特有词汇。

### B. LLM Zero-Shot：指令遵循与常识偏差
*   **典型错误**：输出格式错误（如未按要求输出 JSON/Label），或将模糊文本归类为“其他”。
*   **原因**：0.5B 小模型在零样本下难以严格遵循复杂的分类指令，且其预训练分布与 TNEWS 的新闻分类体系存在偏差。

### C. LLM SFT (LoRA)：过拟合与泛化不足
*   **现象**：在训练集表现优异，但在处理未见过的简短歧义文本时，偶尔会出现“过度推理”，即强行将无关文本关联到某个类别。
