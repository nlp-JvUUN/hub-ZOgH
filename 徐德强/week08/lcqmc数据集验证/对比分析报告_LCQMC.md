## 一、BERT 方案对比（判别式）

### 1.1 训练曲线

| Epoch | Cosine Loss | Cosine Val Acc | Triplet Loss | Triplet Val Acc | CrossEncoder Loss | CrossEncoder Val Acc |
|-------|------------|----------------|--------------|-----------------|-------------------|---------------------|
| 1 | 0.1948 | 76.61% | 0.0280 | 80.41% | 0.3163 | 80.89% |
| 2 | 0.1566 | **78.43%** | 0.0160 | **81.37%** | 0.2121 | **82.39%** |

### 1.2 关键差异：CrossEncoder 又是第一，但 LCQMC 上差距缩小

```
BQ Corpus（金融）：CrossEncoder 87.24% vs BiEncoder Cosine 84.80% → 差距 2.44pp
LCQMC（通用）：    CrossEncoder 82.39% vs BiEncoder Triplet 81.37% → 差距 1.02pp
```

LCQMC 上差距缩小的原因：
- **数据量大掩盖结构优势**：238K 数据下 BiEncoder 充分训练，弥补了部分交互不足
- **问句更短更直接**：LCQMC 平均 10.9 字（BQ 13.9），短句下 BiEncoder 压缩损失更小
- **通用领域语义清晰**：通用问答的正负样本边界更明确，两端编码也能区分

### 1.3 Cosine vs Triplet 对比

| 维度 | Cosine | Triplet | 差距 |
|------|--------|---------|------|
| Accuracy | 78.43% | 81.37% | **+2.94pp** |
| F1 | 78.40% | 81.36% | **+2.96pp** |
| AUC | 0.8532 | **0.8937** | **+0.0405** |
| 最优阈值 | 0.79 | 0.78 | 持平 |

**在 LCQMC 上，TripletLoss 明显优于 CosineEmbeddingLoss**——这与 BQ 相反（BQ 上差距仅 0.2pp，持平）。原因：

1. **Triplet 的三元组数量 = 正样本数（~138K）**，LCQMC 的正样本绝对量 ≈ BQ 全量数据的两倍
2. **LCQMC 正样本多样性强**：同义问法覆盖更多表达方式，TripletLoss 的"相对排序"视角优势被放大
3. **Cosine 在 78-84% 区间平台**：仅用句对标签直接优化，238K 数据下 AUC 仅 0.85，说明正负样本边界在余弦空间中不够清晰

## 二、LLM 方案对比（生成式）

### 2.1 方法对比

| 方法 | 模型 | Accuracy | F1(正例) | Precision | Recall | parse_fail |
|------|------|----------|----------|-----------|--------|-----------|
| Few-shot | DeepSeek | **91.25%** | **91.05%** | 93.19% | 89.00% | 0/400 |
| Zero-shot | DeepSeek | **89.75%** | **89.12%** | 94.92% | 84.00% | 0/400 |
| SFT LoRA | Qwen2.5-0.5B | **84.50%** | **84.10%** | — | — | 0/200 |

### 2.2 零样本 vs 少样本

Few-shot 比 zero-shot 高 **+1.50pp（Acc）+1.93pp（F1）**，提升主要来自 Recall（84%→89%，+5pp）：

| 指标 | Zero-shot | Few-shot | Δ |
|------|-----------|----------|---|
| Precision | **94.92%** | 93.19% | -1.73pp |
| Recall | 84.00% | **89.00%** | **+5.00pp** |
| F1 | 89.12% | **91.05%** | +1.93pp |

Few-shot 的示例（6 对）帮助模型更准确地判断"有歧义的相似"案例，显著降低了假阴性（把相似判为不相似）。

### 2.3 与 BQ 的鲜明对比

```
BQ Corpus（金融）：
  6. LLM Zero-shot  58.29%  ← 金融术语不熟，随便猜
  5. LLM Few-shot   70.10%  ← 示例给了金融样本，勉强及格
  4. SFT LoRA       80.00%  ← 微调后能用

LCQMC（通用）：
  1. LLM Few-shot   91.25%  ← 通用问答全覆盖，直接最高
  2. LLM Zero-shot  89.75%  ← 零样本也接近 90%
  3. SFT LoRA       84.50%  ← 反而低于 LLM API，因为小模型能力有限
```

核心结论：**通用领域选 LLM API，垂直领域选微调 BERT/SFT**。

## 三、BERT vs LLM 对比分析

### 3.1 为什么在 LCQMC 上 LLM 反超 BERT？

| 维度 | BERT（4层） | LLM（DeepSeek/SFT） |
|------|-----------|-------------------|
| 最高 Accuracy | **82.39%（CrossEncoder）** | **91.25%（Few-shot）** |
| 最低 Accuracy | 78.43%（Cosine） | 84.50%（SFT LoRA） |
| 胜者 | — | **LLM 全线碾压** |

**核心原因：通用问答的多样性更适合 LLM 的预训练知识。**
```
LCQMC 通用问答特点 → 话题广泛、表达多样
  "英雄联盟什么英雄最好" vs "英雄联盟最好英雄是什么"    ← 句式变换
  "怎么取消自动续费"     vs "如何关闭自动扣款"         ← 同义改写
  "王者荣耀怎么上分"     vs "LOL 怎么上分"            ← 跨游戏类比

BERT 判别式的局限 → 238K 数据仍不够覆盖所有表达变体
  4 层 BERT 只有 45.6M 参数
  从零学习"取消=关闭""续费=扣款"需要大量共现样本
  结果：78-82%，4 层参数卡住了天花板

LLM 生成式的优势 → 预训练阶段见过海量通用问答
  DeepSeek 的训练语料包含大量社区问答、百科对话
  "取消自动续费"≈"关闭自动扣款"这类对应关系已经学会
  零样本 89.75%，加 6 条示例直接 91.25%
```

### 3.2 SFT LoRA 为什么会卡在 84.50%？

```
LLM 能力分层：
  DeepSeek API（大规模，预训练充分）→ 91.25%  ← 最优
  Qwen2.5-0.5B（小模型，465M）      → 84.50%  ← 次优
  BERT 4层（小模型，45.6M）          → 82.39%  ← 第三
```

**SFT LoRA 84.50% > CrossEncoder 82.39% 的原因：**
- 虽然 Qwen 只有 0.5B，但预训练已经比 BERT 学到更多语义知识
- 20K 的 LoRA 微调只需要"激活"已有知识，不需要从头学
- 模型容量 465M > 45.6M（10 倍），即使 LoRA 只更新部分参数，基数仍然更大

**SFT LoRA 84.50% < DeepSeek 91.25% 的原因：**
- 465M vs DeepSeek 数百亿参数，能力天花板不在一个量级
- Qwen 预训练数据的多样性和质量不及 DeepSeek
- LoRA 只在 20K 数据上微调，无法补足预训练阶段的差距

### 3.3 代价差异

| 维度 | BERT（4层） | DeepSeek API | SFT LoRA |
|------|-----------|-------------|----------|
| 训练时间 | 1-2h GPU | **0**（无需训练） | 0.8h GPU |
| 推理速度 | **6ms/对** | 700ms/条（慢 100×） | 300ms/条 |
| 推理成本 | 0（本地） | 按 token 计费 | 0（本地） |
| 精度 | 78-82% | **89-91%** | 84.50% |

### 3.4 LCQMC 场景的方法选型指南

| 实际场景 | 推荐方法 | 理由 |
|---------|---------|------|
| 通用问答匹配，精度优先 | **LLM Few-shot → 91.25%** | 最高精度，API 即用 |
| 不能联网，需本地推理 | **SFT LoRA → 84.50%** | 本地可控，超越 BERT |
| 需要向量检索 | **BiEncoder Triplet → 81.37%** | 唯一可向量化的方法 |
| 高并发、低延迟 | **CrossEncoder → 82.39%** | 6ms/对，本地运行 |
| 零成本快速验证 | **LLM Zero-shot → 89.75%** | 无需训练，秒级出数 |

## 四、LCQMC vs BQ Corpus 全方法对比

| 方法 | BQ（金融 69K） | LCQMC（通用 238K） | Δ |
|------|--------------|------------------|---|
| BiEncoder Cosine | **84.80%** | 78.40% | **-6.40pp** |
| BiEncoder Triplet | 84.62% | 81.36% | -3.26pp |
| CrossEncoder | **87.24%** | 82.37% | **-4.87pp** |
| SFT LoRA | 80.00% | **84.10%** | **+4.10pp** |
| LLM Zero-shot | 58.29% | **89.12%** | **+30.83pp** |
| LLM Few-shot | 70.10% | **91.05%** | **+20.95pp** |

**跨数据集发现：**

1. **BERT 在金融领域比通用领域高 3-6pp**——金融问句模式化，句式固定，BERT 更容易学到规律
2. **LLM 在通用领域比金融领域高 20-30pp**——DeepSeek 的预训练数据覆盖了大量通用问答
3. **SFT 逆势上涨 4pp**——Qwen2.5-0.5B 虽然小，但在通用数据微调后能泛化到验证集；金融领域则因术语冷门表现受限
4. **两个数据集各有冠军方法**：BQ（CrossEncoder 87.24%）、LCQMC（LLM Few-shot 91.25%）

## 五、总结

### 最终排名（LCQMC）

```
🥇 LLM Few-shot (DeepSeek)   91.25%
🥈 LLM Zero-shot (DeepSeek)  89.75%
🥉 SFT LoRA (Qwen2.5-0.5B)   84.50%
4   CrossEncoder              82.39%
5   BiEncoder Triplet         81.37%
6   BiEncoder Cosine          78.43%
```

### 关键结论

1. **通用问答领域，LLM API 碾压一切**——零样本 89.75%，比 BERT 方案高 7-11pp
2. **TripletLoss 在 LCQMC 上首超 Cosine**（+2.96pp F1），大正样本量 + 多样化表达放大了 Triplet 优势
3. **CrossEncoder 继续领先 BERT 方案**但差距收窄（1-4pp vs BQ 的 2.5pp）
4. **SFT LoRA 在通用领域有价值**（84.50%），但受 0.5B 小模型能力天花板限制
5. **领域决定方法选型**：垂直封闭域→ BERT/CrossEncoder；开放通用域→ LLM API
