## 二、三版 RAGAS 对比

```
指标                    原生版(基线)    原生版(+QR)    LangChain版      QR提升
────────────────────────────────────────────────────────────────────────────
Faithfulness（忠实度）     0.661          0.689          0.678        +4.2%
Answer Relevancy（相关性） 0.337          0.477          0.363       +41.5%
Context Precision（精确率）0.429          0.437          0.497        +1.9%
Context Recall（召回率）   0.512          0.490          0.462        -4.3%
```

### 关键发现

1. **查询改写效果显著**：Answer Relevancy 从 0.337 提升至 0.477（+41%），是当前性价比最高的优化手段
2. **改写有取舍**：Recall 微降 4.3%，改写聚焦后遗漏了一些相关上下文，但 Faithfulness 和 Precision 都有所提升，整体收益为正
3. **原生版 vs LangChain 版**：典型的 Precision-Recall 权衡
   - 原生版混合检索覆盖面大 → Recall 更高（0.490 vs 0.462）
   - LangChain 单路检索噪音少 → Precision 更高（0.497 vs 0.437）
   - 两者 Faithfulness 相当（0.689 vs 0.678），差距不大
4. **综合最优**：原生版 + 查询改写 + semantic 分块

## 三、消融实验（10 题检索指标）

实验矩阵：3 种分块策略 × 3 种检索方式

```
策略            检索方式       Hit@4    MRR
────────────────────────────────────────────
semantic       vector_only    1.000   1.000    ← 最优
semantic       hybrid         1.000   1.000
fixed          vector_only    1.000   1.000
fixed          hybrid         1.000   0.950
hierarchical   vector_only    1.000   0.933
hierarchical   hybrid         1.000   0.950
semantic       bm25_only      0.700   0.600
fixed          bm25_only      0.700   0.625
hierarchical   bm25_only      0.700   0.525
```

### 关键发现

1. **分块策略影响有限**：三种策略在向量检索下 Hit Rate 均为 100%，semantic MRR 最高
2. **vector 碾压 bm25**：BM25 单独在古典小说场景只有 70% 命中率，远低于向量检索，hybrid 几乎全靠 vector 贡献
3. **hierarchical 未达预期**：父子块结构在消融中反而比 semantic 差（MRR 0.933 vs 1.0），父块过大导致检索边界模糊
4. **固定大小分块表现不差**：fixed 和 semantic 在 Hit Rate 上平手，仅 MRR 略低（0.95 vs 1.0）

### 推荐配置

| 组件 | 推荐 | 原因 |
|------|------|------|
| 分块策略 | semantic | MRR 最高，语义完整 |
| 检索方式 | hybrid | 保留 BM25 兜底（对精确关键词仍有价值） |
| 查询改写 | 开启 | 相关性提升 41%，代价可接受 |
| Reranker | 未启用 | 预留接口，可在需要时加载 |

## 四、不足与后续方向

### 当前局限

| 问题 | 影响 | 严重程度 |
|------|------|---------|
| Context Precision < 0.5 | 检索结果含大量噪音，LLM 需从长上下文中筛选 | 中 |
| Context Recall < 0.5 | 约一半有用内容未被检索到 | 中 |
| Answer Relevancy < 0.5 | 即使开启改写，约半数回答未充分切题 | 中 |
| 8/20 题拒绝回答 | 跨书对比、开放性问题超出知识库覆盖 | 低（部分为正确行为） |

### 改进路线

1. **Cross-Encoder Reranker**（最高优先级）：预期 Context Precision 提升至 0.55+，Faithfulness 跟随提升
2. **BGE 更大模型**（如 bge-large-zh）：预期 Recall 和 Precision 同时提升，但推理时间和显存需求增加
3. **知识库扩展**：补充原文缺失段落、增加人物/事件知识卡片作为 fallback
4. **多轮改写**：对跨书对比问题做拆分检索（如先查 A 书，再查 B 书，最后合并对比）
