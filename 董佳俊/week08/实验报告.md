## 四、关键发现

### 4.1 CrossEncoder 在均衡数据上最优

与 AFQMC 的结论不同——当数据正负均衡时，CrossEncoder 的全层交互优势充分发挥，在两个数据集上均排名第一。AFQMC 上 CrossEncoder 被类别不均衡拖累（退化为全预测负类），这两个数据集正好验证了"均衡数据 + CrossEncoder = 最优"的结论。

### 4.2 CosineLoss 持续优于 TripletLoss

- LCQMC：CosineLoss 领先 TripletLoss **4.76 个 F1 点**
- BQ Corpus：CosineLoss 领先 TripletLoss **0.28 个 F1 点**

根因：TripletLoss 的三元组数量受限于正样本数（平衡采样后约 10K），训练信号偏少。train_loss 极低（LCQMC: 0.018）说明 margin 很快"打满"。

TripletLoss 阈值偏高（0.91 / 0.59），因为 TripletLoss 只约束相对关系，不约束绝对相似度。

### 4.3 数据集难度差异

BQ Corpus 的 F1 整体高于 LCQMC（最高 0.773 vs 0.754）：
- BQ Corpus 是银行客服领域，问句语义边界清晰，同义改写相对规范
- LCQMC 来自百度知道，是开放域通用问句，表述多样性大，同义异词现象更多

## 五、Bad Case 分析（CrossEncoder）

### 5.1 LCQMC 错误分布

| 错误类型 | 数量 | 占比 |
|---------|-----:|-----:|
| FP 假阳性（预测相似、实际不同） | 1,384 | 64% |
| - 高置信度错误 | 1,064 | |
| - 临界错误 | 320 | |
| FN 假阴性（预测不同、实际相似） | 774 | 36% |
| - 高置信度错误 | 527 | |
| - 临界错误 | 247 | |
| **总计** | **2,158** | |

**语言特征指纹：**

| 特征 | FP（假阳性） | FN（假阴性） |
|------|----------:|----------:|
| 字符 Jaccard | **0.710** | 0.527 |
| 长度差均值 | 1.2 字 | 2.5 字 |

> FP 的 Jaccard 高达 0.71 —— "词汇高度重叠但语义不同"是 LCQMC 错误的主因。

**典型 FP 案例（CrossEncoder 判相似、实际不同）：**
```
score=0.992 | "三模连接是2，4版块" || "三模连接是3，5版块"
score=0.992 | "做图软件的素材去哪里找?" || "做图软件，素材去哪里找"
score=0.988 | "腿皮肤为什么有蛇皮纹?" || "腿皮肤为什么像蛇皮"
```

**典型 FN 案例（CrossEncoder 判不同、实际相似）：**
```
score=0.005 | "从哪里可以观看电影/电视剧/视频/下载地址" || "420_从哪里可以看电视电视剧视频看电影"
score=0.006 | "有诗朗诵和合唱的串词" || "诗朗诵合唱的串词"
score=0.009 | "初中化学元素化合价口诀" || "化学元素化合价口诀"
```

### 5.2 BQ Corpus 错误分布

| 错误类型 | 数量 | 占比 |
|---------|-----:|-----:|
| FP 假阳性 | 945 | 48% |
| FN 假阴性 | 1,009 | 52% |
| **总计** | **1,954** | |

**语言特征指纹：**

| 特征 | FP（假阳性） | FN（假阴性） |
|------|----------:|----------:|
| 字符 Jaccard | 0.282 | **0.154** |
| 长度差均值 | 5.5 字 | 7.0 字 |

> BQ Corpus 的 FN Jaccard 仅 0.154 —— 银行客服的同义表达几乎不共享字符，"存款有保障吗"="不知道安全吗"这种极端的同义改写是最大挑战。

**典型 FP 案例：**
```
score=0.980 | "为什么我没额度需要怎么操作" || "怎么操作需要额度"
score=0.975 | "为什么说我不符合条件" || "怎样不符合条件"
```

**典型 FN 案例：**
```
score=0.035 | "信用卡的账单可以分几次还吗" || "一次性还清还是可以分期还"
score=0.044 | "微信支付信用卡有什么优点" || "QQ钱包支付"
score=0.052 | "我只要还款明细" || "能不能查到，一次性还清吗"
```

### 5.3 两个数据集的 Bad Case 对比

| 维度 | LCQMC | BQ Corpus |
|------|-------|-----------|
| 主导错误类型 | **FP**（64%） | FN/FP 均衡（52%/48%） |
| FP 根因 | 词汇重叠高但语义不同（Jaccard=0.71） | 语义判断不准确（Jaccard=0.28） |
| FN 根因 | 换词改写（Jaccard=0.53） | 完全换说法（Jaccard=0.15） |
| 优化方向 | 增大 margin + 难负样本挖掘 | SimCSE 对比学习 + 增加层数 |

## 六、方法选型建议

| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| **大规模向量检索（RAG Recall）** | BiEncoder + CosineLoss | 向量可预计算，F1 接近 CrossEncoder（差 < 2%） |
| **精排（Reranker）** | CrossEncoder | F1 最高，两句全层交互，精度优势明确 |
| **快速原型验证** | BiEncoder + TripletLoss | 训练最快（batch 数少 50%），但精度略低 |
| **数据不均衡场景** | BiEncoder + CosineLoss | 比 CrossEncoder 更鲁棒（AFQMC 经验） |
| **同义异词严重场景** | CrossEncoder + SimCSE | BQ Corpus 启示：需要对比学习增强语义理解 |

## 七、消融与后续方向

| 变量 | 当前值 | 建议尝试 |
|------|--------|---------|
| BERT 层数 | 4 | 8 / 12 层，预计 F1 提升 3-8 个点 |
| 训练 epoch | 1 | 3 epoch，注意观察 val_loss 拐点 |
| 训练数据量 | 20K 采样 | 全量数据（LCQMC 239K，BQ 69K） |
| 池化策略 | mean | cls / max 对比 |
| margin | 0.3 | 0.1 / 0.5 消融 |
| 两阶段级联 | 无 | BiEncoder 召回 Top-K + CrossEncoder 精排 |
