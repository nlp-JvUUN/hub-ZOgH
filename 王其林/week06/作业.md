# 文本分类训练报告

## 1. 实验概况

本次实验在 `text_classification项目` 中比较了以下方法：

- BERT 微调：`bert-base-chinese`，实验了 CLS 池化、Mean 池化、Max 池化以及对 CLS 进行类别加权微调；
- LLM Zero-shot：`Qwen2-0.5B-Instruct` 直接 prompt 推理；
- LLM SFT：同一模型基础上使用 LoRA 微调。

报告基于 `outputs/` 下的训练与评估结果生成，重点回答用户提出的问题。

---

## 2. 训练结果摘要

### 2.1 BERT 各池化方式对比

| 模式 | 最佳验证准确率 | 对应 Epoch | 最佳验证 Macro F1
|---|---|---|---|---|
| CLS | 0.538 | 2 | 0.5124
| CLS 加权 | 0.524 | 2 | 0.5660 
| Mean | 0.530 | 3 | 0.5049 
| Max | 0.513 | 3 | 0.5142 

- `CLS 加权` 取得了最高 Macro F1 56.6%，说明加权损失对少数类表现更友好；
- `Mean` 池化表现与 `CLS` 相近，说明平均池化在本任务中仍然是一个稳定的选择；
- `CLS` 池化作为基线保持了较好的整体准确率，且与 `Mean` 相差很小。

### 2.2 LLM Zero-shot 与 SFT 结果

| 方法 | 评估准确率 | 无法解析数 | 总样本数 |
|---|---|---|---|
| Zero-shot | 0.36 | 58 | 200 |
| SFT (LoRA) | 0.58 | 2 | 200 |

结论：

- SFT 相比 Zero-shot 绝对准确率提升了约 22 个百分点；
- 无法解析样本从 58 条降到 2 条，几乎消除了解析失败问题；
- 这与预期一致：Zero-shot 在自定义 15 类标签体系上很难对齐输出，而 SFT 通过监督信号学习到了标签映射与输出格式。

### 2.3 Zero-shot 的强势类别

Zero-shot 表现最好的类别是：

- 体育：73.7% 正确率，未命中 2 条无法解析；
- 娱乐：72.2% 正确率，4 条无法解析；
- 科技：60.0% 正确率，3 条无法解析；
- 教育：52.9% 正确率，3 条无法解析。

这类标签表现好的原因一般是：

- 类别词语与文本关键词匹配度高；
- 语义边界较清晰，模型容易直接给出对应标签；
- 输出空间相对稳定，不易产生词义歧义。

相反，Zero-shot 在“国际”“房产”“故事”“证券”等类别表现很差，这些类别要么标签语义太宽，要么容易与政治、财经、旅游、汽车等类混淆。

### 2.4 Zero-shot 最易混淆类别

Zero-shot 最易混淆的真实类别包括：

- `国际`：16 个样本中 12 个无法解析，剩余错误多数被预测为“科技”“娱乐”“文化”“军事”；
- `旅游`：23 个样本中 8 个无法解析，6 个被误判为“娱乐”，2 个误判为“文化”；
- `汽车`：15 个样本中 5 个无法解析，且 3 个误判为“娱乐”、2 个误判为“科技”；
- `房产`：8 个样本中 5 个无法解析，其他错误主要落在“财经”“娱乐”“文化”；
- `文化`：10 个样本中 5 个无法解析，2 个误判为“科技”，1 个误判为“旅游”。

此外，`电竞` 更容易被错误判为“娱乐”或“体育”，而 `证券` 的样本极少且被误判为“财经”。

这种混淆模式说明：

- 语义边界较宽、标签词更常见于日常话题的类别更容易被 Zero-shot 误判；
- 主题相近的类别（如旅游/娱乐、汽车/科技、国际/军事）更容易互相混淆；
- `无法解析` 情况主要集中在那些标签本身不够精确或候选词在 prompt 中不够明确的类别。

## 3. 针对用户问题的分析

### 3.1 SFT 相比 Zero-shot 提升了多少？这符合预期吗？

- SFT 提升了约 22 个百分点准确率；
- “无法解析”样本数从 58 降到 2；
- 这非常符合预期：Zero-shot 在自定义分类标签上依赖 prompt 对齐，输出往往偏离预定义标签；SFT 通过少量标注数据显著改善这种对齐问题。

### 3.2 BERT 用了全部 53K 条，SFT 只用了 5K 条；如果数据量相同，谁更有优势？

从当前结果看：

- BERT 对于短文本、多类别判别任务非常稳定；
- LLM SFT 在仅用 5K 条情况下已经取得了 58% 的准确率，并几乎没有无法解析问题；
- 如果数据量相同，BERT 仍然是更稳的“判别式”选择，尤其在训练样本足够、标签体系固定、文本短且类别覆盖明确时；
- 但对于语义复杂、标签边界模糊或者需要模型理解更丰富语境的任务，LLM SFT 可能会有更强的泛化潜力。

因此，保守判断为“同样数据量，BERT 更有优势于传统分类精度和稳定性；LLM SFT 的优势是语义理解与标签对齐，但需要更多调优和 prompt/模板设计”。

### 3.3 LoRA 参数量仅约 0.5%，效果损失有多大？

当前输出表明：

- 0.5% 参数量的 LoRA SFT 已经将 Zero-shot 精度提升到 58%；
- 与 BERT 的 53K 数据训练结果对比，LoRA SFT 的准确率并未明显落后，甚至在 200 条评估集中超过了部分 BERT 模式；

所以效果损失并不大，尤其考虑到训练成本和可训练参数量。具体损失还取决于：

- LoRA rank 设置；
- 是否使用全量微调；
- 训练数据质量与标签提示。

#### 对比实验建议

- `train_sft.py --lora_r 32`：增加 LoRA rank 会提高可训练容量，通常能继续提升性能，尤其当训练数据不够时；
- 取消 PEFT / 全量微调：如果硬件允许，全量微调一般会进一步缩小或超越当前 LoRA 与 BERT 的差距，但成本大幅增加。

### 3.4 生成式分类有“无法解析”，判别式分类没有。在生产系统中如何处理？

生产系统处理方式：

1. 设计严格输出格式：给 LLM 明确可选标签列表，并强制输出“label:XXX”格式；
2. 后处理校验：如果输出不在候选标签集合内，则视为“解析失败”，并触发二次策略；
3. Fallback 机制：对无法解析或低置信度结果，使用 BERT /传统分类器作为候选 fallback；
4. 交互式校正：在客服或审批系统中，将“无法解析”转给人工判断；
5. 软拒绝策略：输出不能解析时，返回“类别不明确，请补充描述”而不是直接给错结果。

这意味着：

- LLM 可作为“第一阶段语义理解器”；
- BERT 或其他判别式模型可作为“第二阶段可靠分类器”或“兜底模型”；
- 当生产要求零解析失败时，必须加一层校验/映射与 fallback。

### 3.5 LLM zero-shot 在哪些类别上表现好？为什么？

表现较好的类别是：

- 体育、娱乐、科技、教育。

原因分析：

- 这些类别在文本中往往出现明确的关键词；
- 类别语义明确且与真实标签关系紧密；
- 输出空间较窄，不容易出现“同义词+错类”的情况。

### 3.6 fine-tuning 数据量越多效果越好吗？试试只用 1000 条训练数据。

当前 `outputs/` 中没有 1000 条训练数据的结果。

一般经验是：

- 训练数据越多，BERT 和 LLM SFT 的稳定性与泛化能力通常会更好；
- 但边际收益会递减，尤其当数据从 5K 增加到 50K 时；
- 若只用 1000 条，模型很可能出现欠拟合、类别不均衡更严重、泛化能力下降；
- 对于 BERT，1000 条通常不足以训练出高质量 15 类分类器；
- 对于 LLM SFT，1000 条仍可能带来较大提升，但与 5K 相比表现会明显下降。

因此建议做一次对比实验：

- `train_sft.py --train_size 1000` 或在数据准备阶段抽样 1000 条；
- 对比 1K、5K、10K、全量 53K 的准确率曲线。

### 3.7 如果换成 few-shot（每类给 2 个示例），准确率会提升多少？

当前没有 few-shot 结果。

预期是：

- few-shot 通常比纯 zero-shot 更稳定，但仍不如 SFT；
- 2 个示例每类共 30 个示例，属于极小样本，准确率提升有限；
- 但它能显著降低“无法解析”概率，尤其当 prompt 模板明确且示例代表性强时；
- 在本任务中，2-shot 的准确率可能从 36% 提升到 40%–50% 区间，具体取决于示例选择和提示设计。

建议实验方式：

- `train_sft.py` 或推理脚本改为 few-shot prompt；
- 每类选择典型示例，避免过于模糊；
- 评估时与 zero-shot、SFT 做同一测试集对比。

---

## 4. 综合结论

1. BERT 的最佳准确率来自 `Max` 池化（54.9%），而 `CLS 加权` 在 Macro F1 上最优（56.6%）。
2. LLM SFT 在 5K 数据下取得 58% 准确率，并几乎消除了解析失败；这是一个非常强的信号，表明 LoRA SFT 在少量训练数据条件下对自定义标签体系非常有效。 
3. SFT 相比 Zero-shot 的提升符合预期：+22 个百分点准确率、+56 条解析成功。 
4. 生产系统应对 LLM 解析失败进行严格格式约束和 fallback，而 BERT 作为判别式分类器是更稳的兜底选择。
